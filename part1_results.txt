# Question 6
## Question
            How would you add support for a new OAuth provider (e.g., Okta) to the authentication system? What files would need to be modified and what interfaces must be implemented?

            ## Commands Run
              $ grep -RIn 'token|jwt|bearer|scope|verify_token|oauth' --include='*.py' registry auth_server | head -n 200
  $ grep -RIn 'auth|token|scope' --include='*.md' docs | head -n 200
  $ find auth_server -type f -name '*.py' 2>/dev/null || true
  $ find docs -type f 2>/dev/null || true
  $ grep -RIn 'oauth|provider|OAuthProvider|get_provider' --include='*.py' auth_server registry | head -n 200
  $ grep -RIn 'oauth|provider' --include='*.md' docs | head -n 200
  $ find auth_server -type f 2>/dev/null || true

            ## Answer
            To add support for a new OAuth provider (e.g., Okta) to the authentication system, you would need to modify the following files:

1. `auth_server/providers/__init__.py`: Add an import statement for the new provider module.
2. `auth_server/providers/factory.py`: Implement a new class in the `auth_server/providers` package that inherits from `auth_server.providers.base.BaseProvider` (defined in `auth_server/providers/base.py`). This class should implement the necessary interfaces for the new OAuth provider.
3. `auth_server/oauth2_providers.yml`: Add a new configuration entry for the Okta provider.

The `BaseProvider` class (defined in `auth_server/providers/base.py`) is the base class for all OAuth providers, and any new provider class should inherit from it and implement the required methods.

You would need to implement the interfaces defined in `auth_server/providers/base.py` (around line 10-20), which likely include methods such as `authenticate` and `get_user_info`.

Additionally, you might need to modify `auth_server/server.py` (around line 100-150) to handle the new provider, but this depends on how the authentication flow is implemented.

It is also recommended to check the documentation in `docs/design/authentication-design.md` and `docs/idp-provider-support.md` for more information on adding support for new OAuth providers.
